# Note Templates

Reusable templates for writing high-quality notes in `05-knowledge/`. Every note should pass the future-me test: "Will future-me thank me for this?"

## Quality Gates

Before saving any curated note:

1. Written for a reader who forgot the context?
2. Includes WHY, not just WHAT?
3. Has a concrete example or key insight?
4. Structured for scanning (headings, bullets, tables)?

---

## Concept Template

```markdown
# [Concept Name]

## What It Is

[One-line definition]

## Why It Matters

[What problem it solves, when you'd need it]

## How It Works

[Explanation with examples]

## Key Insight

[The "aha" moment, what makes this click]
```

---

## Tool Template

```markdown
# [Tool Name]

**Category:** [devtools | productivity | infrastructure | etc.]

## What It Does

[Brief description]

## Why We Use It

[What problem it solved, specific to our context]

## When to Reach For It

[Scenarios where this is the right choice]

## Gotchas

- [Things that tripped us up]
```

---

## Pattern Template

```markdown
# [Pattern Name]

## Problem

[What situation triggers this pattern]

## Solution

[The approach]

## Trade-offs

**Pros:** [Why this works]
**Cons:** [When NOT to use it]
```

---

## Decision Template

```markdown
# [Decision]: [Choice Made]

**Date:** YYYY-MM-DD
**Context:** [What prompted this decision]

## Options Considered

1. **[Option A]** - [pros/cons]
2. **[Option B]** - [pros/cons]

## Decision

[What we chose and why]

## Revisit If

[Conditions that would make us reconsider]
```
